Guideline Baseline

  • Liquid Glass is a functional layer for controls and navigation, not general content chrome.
  • Controls should remain legible over content and preserve hierarchy between functional and content layers.
  • Prefer circular or capsule buttons; use rounded rectangles for stacked text buttons.
  • Button centers should be at least 60 pt apart where applicable; large buttons need extra hover padding.
  • macOS accessibility minimum target is 28x28 pt, with 20x20 pt visual minimum; spacing matters as much as size.

Sources queued for Oracle synthesis: Apple HIG, Buttons, Materials/Liquid Glass, Color, Accessibility.

Image Diffing Recommendation

The audit now has a local ImageMagick-backed DevTools primitive at scripts/devtools/image-diff.ts so before/after screenshots can produce numeric receipts and diff masks instead of relying on visual inspection alone. It supports full-image comparison, explicit per-side crops, and receipt-driven target crops for component-level proof.

  • Save masks beside each before/after pair and keep the JSON receipt next to the layout and screenshot receipts.
  • Generate focused overlays that call out rounded-corner deltas, padding shifts, hit-target expansion, row-height changes, popup anchor movement, and material/blur region changes.
  • Record numeric summaries in this log: changed-pixel count, changed-pixel ratio, bounding boxes of differences, per-corner radius evidence, and expected-vs-actual padding/token deltas.
  • Prefer component-cropped diffs for buttons, selected rows, input chrome, footers, popups, and window corners so tiny Tahoe/Liquid Glass regressions are easy to see.
  • Keep raw screenshots, diff masks, and JSON diff receipts under artifacts/liquid-glass/, then link them from each surface row.
