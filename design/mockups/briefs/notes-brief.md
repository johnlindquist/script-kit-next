# Notes screen — design-contract brief

Screen slice for the Notes window (separate NSPanel hosting the markdown
editor). Mockup: `design/mockups/screens/notes/`. Every metric/color claim
below is tagged **MEASURED** (read from source at the cited file:line) or
**GUESS** (inference; must be verified against a live capture/probe before the
tokens ship).

Status: **SHIPPED (2026-07-11, per the Oracle notes-window-html-mockup
review)** — the `--sk-notes-*` tokens are generated by the notes section of
`src/design_contract/mod.rs::checked_in_design_bundle`, backed by three
contract owners:

- `src/notes/window/contract.rs` — app-authored window/titlebar/footer
  chrome + the layout MODEL (28px footer reservation, autosize numbers),
  resolved from the production profile (`NotesWindowStyle::current()`, NOT
  the feature-sensitive `adopted_style()`). `window_ops` (default bounds,
  traffic-light origin) and `render_editor_titlebar` (padding-x) consume the
  same consts.
- `src/components/notes_editor/contract.rs` —
  `ResolvedNotesEditorMetrics` (base font family/size via the SAME
  theme→gpui-component bridge the renderer syncs, line box 20, caret 2×17);
  the vendored Input arithmetic lives ONLY there.
- `src/notes/markdown_highlighting.rs` — `resolved_notes_markdown_styles`
  reads the real gpui-component highlight theme
  (`build_markdown_highlight_theme`) for the `title`,
  `punctuation.special`, and `punctuation.list_marker` captures. No vendored
  visibility change was needed (`SyntaxColors::style` is already `pub`).

A reference capture exists (`screens/notes/reference/notes@2x.png` +
`notes.crop.json`, 350×280@2x). The old `--sk-emulator-draft-notes-*` alias
block is deleted; the mockup consumes exporter output directly.

## 2. Fixture strategy

### Sandbox + open path (verified against `scripts/devtools/notes.ts`)
- **Sandbox DB**: launch the app with `SCRIPT_KIT_TEST_NOTES_DB_PATH=<tmp>/notes.sqlite`
  in the app environment (notes.ts:618-619 builds it; passed to
  `session.sh start` env at notes.ts:110,627). Under this env the Notes
  window also pins **default 350×280 top-right bounds**
  (`window_ops.rs:672-674`) — no saved-bounds contamination.
- **Open**: protocol send `{ "type": "openNotes" }` (notes.ts:115-122).
- **Target**: automation window id `"notes"`, kind
  `AutomationWindowKind::Notes` — MEASURED:
  `src/notes/window/window_ops.rs:748-760`; devtools resolves it with
  `targets.ts inspect --target-kind notes --strict` (notes.ts:310-330).
- **Seed content**: target-scoped batch `{ type: "setInput", text }`
  (notes.ts:568-583). The tool then waits for **two consecutive equal
  window heights** instead of a fixed sleep (`waitForStableNotesHeight`,
  notes.ts:585-596) — reuse that settle.
- **Sandbox guard**: refuse mutation unless runtime state reports
  `storage.testSandbox === true` (notes.ts:634-643) — keep this in the
  capture script.

### `scripts/agentic/design-reference-capture.ts --screen notes` (capture recipe — the 2026-07-11 reference was produced this way)
Extend the existing maps (design-reference-capture.ts:26-33):

```ts
// DEFAULT_OUT
notes: "design/mockups/screens/notes/reference/notes@2x.png",
// CAPTURE_TARGET — Notes has its own automation kind
notes: { type: "kind", kind: "notes" },
```

and add a screen branch (launch must carry the sandbox env — unlike the
main/actions/confirm screens this one needs a custom session env):

```ts
if (screen === "notes") {
  // 1. Session start env: SCRIPT_KIT_TEST_NOTES_DB_PATH=<mkdtemp>/notes.sqlite
  //    (pins default 350×280 bounds; window_ops.rs:672-674)
  await driver.send({ type: "openNotes" });
  await waitForTargetKind("notes"); // targets.ts --target-kind notes --strict
  await driver.batch({
    target: { type: "kind", kind: "notes", index: 0 },
    commands: [{ type: "setInput", text: NOTES_FIXTURE_MARKDOWN }],
    options: { stopOnError: true, timeout: 8000 },
  });
  await waitForStableNotesHeight(); // notes.ts:585-596 pattern
  // 2. Keep the mouse OFF the window: fixture is the REST state (footer 0.5,
  //    title 0.7). Do NOT set SCRIPT_KIT_TEST_NOTES_HOVERED.
  // 3. captureScreenshot({ target: { type: "kind", kind: "notes" } })
}
```

`NOTES_FIXTURE_MARKDOWN` must equal the mockup content exactly (8 lines):

```
# Design Contract Notes

Track every painted value in
the Notes window.

- Titlebar 36 px
- Editor 16 px mono, 20 px line
- Footer buttons hug
```

Titlebar then shows the note title "Design Contract Notes" (title = first
line — `selected_note_title`, `src/notes/window/render_editor.rs:4-28`).
Caveats: caret blink phase at capture time (divergence entry), and the save
status may show the 1500 ms "✓" flash if captured too soon — settle > 1.6 s
after the last `setInput` before capturing (window.rs:170 `SAVED_FLASH_MS`).
