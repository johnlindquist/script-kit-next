# One-Key Gesture Grammar

The main hotkey carries all memory-layer intent through physical gesture: tap (closed) opens the launcher on key-down; tap (open) toggles launcher <-> Day Page with in-flight query text carried over; hold (~250ms) deepens the window into the Day Page with push-to-talk dictation hot (release commits to today's page); double-tap opens Agent Chat from anywhere. Cmd+Enter (the established agent gesture) and Esc (the only dismiss) are the only other keys in the grammar. The disambiguation rule is "always open, then deepen": the launcher renders on key-down in every case, and hold/double-tap escalate the already-open window, so no gesture ever waits on another and tap latency is never taxed.

We chose this over a Tab-based universal grammar (conflicts with editor Tab formatting; Tab AI is deprecated in favor of Cmd+Enter), over per-surface shortcuts (users should not learn new chords), over a launcher fallback row that keeps unmatched queries as notes (users do not type into the launcher expecting capture without expressed intent — semicolon tags and the hold gesture carry that intent), and over wait-to-disambiguate gesture detection (any delay on tap breaks the instant-launcher habit loop).

Constraints: surfaces morph inside one window — windows are never swapped, because swaps flash and break the instant feel (the main window already morphs content via `MainWindowMode` and in-window prompt surfaces). Tap-while-open replaces tap-to-dismiss, which is a deliberate habit break; Esc becomes the only dismiss. The morph must be geometrically calm: the input row keeps its pixels and the window grows a body. Detailed state machine, timings, and open questions live in `docs/specs/gesture-grammar.md`.
