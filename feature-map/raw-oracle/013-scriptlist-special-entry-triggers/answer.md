013 ScriptList Special Entry Triggers / First-character Route Handoffs Executive Summary Feature 013 is a narrow ScriptList input-change router. It is not a general parser, not source-filter handling, not menu-syntax handling, and not power-user command parsing. The owner is ScriptListSpecialEntry in src/app_impl/filter_input_core.rs, with five committed variants: FileSearchMini, AcpSlashPicker, AcpMentionPicker, QuickTerminal, and ActionsHelp. The classifier accepts only these shapes: ~, ~/..., /, @, >, and ?; everything else falls through to ordinary ScriptList behavior or adjacent feature owners. scriptlist-special-triggers-atl… The runtime handoff happens in ScriptListApp::handle_filter_input_change in src/app_impl/filter_input_change.rs. Once the classifier returns a variant, the handler logs script_list_special_entry_routed, dispatches the exact helper for that variant, and returns before normal menu-syntax mode update continues. The dispatch table is explicit: mini File Search calls open_file_search_view(query, FileSearchPresentation::Mini, cx), / calls open_tab_ai_acp_with_slash_picker, @ calls open_tab_ai_acp_with_mention_picker, > calls open_quick_terminal(None, cx), and ? calls toggle_actions(cx, window) only when has_actions() is true. scriptlist-special-triggers-atl… The hardest invariant is the negative space: /tmp, @browser, >deploy -- prod, :, +, !, #, capture syntax, and calendar/task menu syntax must not be routed through this classifier. Tests already pin examples for path text, mention-like literals, power syntax, and source/menu prefixes. scriptlist-special-triggers-atl… The product split is also load-bearing. / and @ hand off to Agent Chat / ACP, while > hand off opens Quick Terminal, a PTY-backed wrapper with terminal-owned Tab/Escape behavior, terminal-specific Cmd+Enter apply-back, Cmd+W/native-footer close, and compact launcher sizing. Quick Terminal is not ACP Chat. scriptlist-special-triggers-atl… scriptlist-special-triggers-atl… What Users Can Do Users can type ~ in the main launcher to jump into mini File Search rooted at home. Bare ~ normalizes to ~/; ~/src stays ~/src. This is a route handoff into FileSearchPresentation::Mini, not a literal ScriptList search. The docs also state that the ~ trigger opens mini File Search with directory rows seeded before first paint, preventing a zero-row flash while the async directory stream catches up. scriptlist-special-triggers-atl… scriptlist-special-triggers-atl… Users can type / by itself in the main launcher to open Agent Chat with the ACP slash picker. The helper stages tab_ai_harness_script_list_trigger = Some('/'), opens ACP, and for embedded ACP defers picker opening through schedule_embedded_acp_picker_open. Contract tests pin that the trigger must be staged before ACP opens and that picker opening must be deferred rather than done inline. scriptlist-special-triggers-atl… scriptlist-special-triggers-atl… Users can type @ by itself in the main launcher to open Agent Chat with the ACP mention/context picker. This is the same ScriptList-to-ACP pattern as /, but with tab_ai_harness_script_list_trigger = Some('@'); once inside ACP, the mention picker owns context resources, inline tokens, provider filtering, and portal entries. scriptlist-special-triggers-atl… scriptlist-special-triggers-atl… Users can type > by itself to open Quick Terminal. This route calls open_quick_terminal(None, cx), switches current_view to AppView::QuickTerminalView, clears text-input focus by setting focused_input = FocusedInput::None, requests FocusTarget::TermPrompt, refills the warm PTY pool, and notifies the UI. scriptlist-special-triggers-atl… Users can type ? by itself to open actions/help only when the current ScriptList state has actions. The route is guarded by has_actions(): when false, the special-entry branch consumes the route and returns without opening the actions dialog. scriptlist-special-triggers-atl… Users can keep typing ordinary query text without accidentally triggering these routes. Examples pinned as non-routes include /tmp, @browser, foo, >deploy -- prod, :, +, !, #, ;todo Renew passport tomorrow, todo: Renew passport tomorrow, and cal: Lunch next friday. scriptlist-special-triggers-atl… Core Concepts Special entries are route handoffs, not search grammar. ScriptListSpecialEntry owns only the small list of first-character handoffs. set_menu_syntax_mode_from_filter is a separate menu-syntax owner and runs after the special-entry block only when no special route matched. scriptlist-special-triggers-atl… scriptlist-special-triggers-atl… Transient trigger text is not destination content. The transient triggers are exactly ~, /, @, >, and ?, as pinned by is_transient_script_list_trigger. These are control gestures typed through the shared filter, not durable ScriptList search terms. Returning from ACP/terminal/file-search flows must not leave a stale one-character launcher query as if the user had searched for it. scriptlist-special-triggers-atl… Tilde is the one special entry that accepts a prefix form. should_enter_file_search_from_script_list returns true for new_text == "~" or new_text.starts_with("~/"). Bare / and bare @ route to ACP picker flows. Non-bare strings like /tmp and @browser are ordinary ScriptList text at this layer.
