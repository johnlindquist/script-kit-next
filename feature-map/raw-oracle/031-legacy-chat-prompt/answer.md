Capability summary The legacy chat() prompt is Script Kit GPUI’s SDK-driven conversational prompt surface. It is exposed from the TypeScript SDK as chat(options) in scripts/kit-sdk.ts and rendered in Rust as prompts::ChatPrompt under AppView::ChatPrompt. It is a Raycast-style inline chat UI with message history, markdown turn rendering, streaming assistant responses, model selection, actions, image attachment support, setup-card behavior for missing providers, and optional persistence to the AI chat database. This surface is distinct from the ACP Chat SDK APIs: Legacy chat uses SDK/protocol message types such as chat, chatMessage, chatStreamStart, chatStreamChunk, chatStreamComplete, chatSetError, chatClearError, chatClear, and chatSubmit. ACP Chat uses the separate ai* capability surface and ACP-specific telemetry/message names. The bundle’s lat.md/acp-chat.md explicitly frames Mini AI as allowed to remain ChatPrompt-backed while Full ACP has its own handoff and state contracts. Legacy chat is prompt-centered: it opens a ChatPrompt entity inside the launcher/main window. ACP Chat is thread/session-centered and belongs to the newer AI/ACP runtime. Primary users: Script authors calling chat() from the TypeScript SDK. Human users interacting with the chat prompt UI. Built-in Mini AI / inline AI paths that still use ChatPrompt. Test/automation agents validating protocol, state, visual layout, callbacks, and Mini AI contracts.
