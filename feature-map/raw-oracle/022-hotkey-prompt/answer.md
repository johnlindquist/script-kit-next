022 Hotkey Prompt / hotkey() Executive Summary Feature 022 has two related but separate systems: SDK-visible hotkey(placeholder?) prompt This is exposed in the TypeScript/Bun SDK and returns a Promise<HotkeyInfo>, but the GPUI host currently treats it as unsupported / coming soon. The SDK implementation explicitly warns that hotkey() is not implemented and installs a mock Escape-key fallback payload. The Rust prompt handler receives PromptMessage::HotkeyComingSoon, logs a structured warning, and shows a coming-soon toast. There is no proven full AppView::HotkeyPrompt, no real capture UI, and no proven host-side response envelope for real user input. Implemented shortcut recorder for assigning launcher/app shortcuts This is a real product surface. It is a compact native popup used from action menus to assign or remove persistent command shortcuts. It writes to ~/.scriptkit/config.ts through update-config-shortcut.ts / remove-config-shortcut.ts, updates the live global hotkey registry on save, refreshes visible script data in relevant paths, and uses source-audit tests to keep shortcut assignment tied to config-backed command IDs rather than legacy shortcuts.json. The critical operator boundary is this: SDK hotkey() is a transient prompt API and is currently stubbed; the shortcut recorder is a persistent app shortcut-assignment UI and is implemented. Do not use the implemented recorder as proof that SDK hotkey() works.
