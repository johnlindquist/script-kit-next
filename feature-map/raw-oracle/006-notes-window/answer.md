Notes is explicitly documented as a separate floating host with its own editor state, actions UI, list/trash modes, and reusable embedded ACP surface. notes-window-feature-map User Stories As a user, I can open Notes as a separate floating window that does not behave like a launcher panel clone. As a user, I can create a new note, create a note from clipboard, type Markdown, preview Markdown, use formatting helpers, and rely on debounced save. As a user, I can browse/switch notes, search within the current note, search across notes, pin notes, jump through note history, and sort notes. As a user, I can delete notes into Trash, view Trash, restore notes, permanently delete notes, and empty Trash. As a user, I can copy a note as Markdown, copy a deeplink, create a Markdown quicklink, and copy/export HTML-style output. As a user, I can send the current note/selection/cart to Agent Chat inside the Notes window without opening the main launcher ACP. As a user, I can switch between Notes editor mode and Notes-hosted ACP mode while keeping the embedded ACP conversation/draft when reuse is intended. As a user, I can open Notes-hosted ACP actions, change agent/model/history, and have actions apply to the originating embedded ACP view rather than a replaced/stale view. As a user, I can open local ACP History from Notes-hosted ACP and attach/cancel through the same staged replacement contract used elsewhere. As a user, I can use Notes Browse as an attachment portal to search active notes, preview content, attach a selected note, or cancel cleanly. As an automation user, I can target Notes as {kind:"notes"} and not accidentally hit the main window. The targeting test registers main and Notes separately, resolves Notes by kind/title, and verifies Notes removal from the registry. notes-window-feature-map State Model Window/surface modes: NotesSurfaceMode::Notes: editor/list/trash/search/action/browse overlays render inside the floating Notes host. NotesSurfaceMode::Acp: embedded Notes-hosted ACP replaces the editor body; host intercepts Escape/Cmd+K/Cmd+W before propagating to ACP. NotesViewMode::AllNotes vs NotesViewMode::Trash: active list source and available actions change. Overlay/dialog states: command bar/actions panel, note switcher/browse panel, search bar, format toolbar, confirm dialog, focus mode, ACP actions popup, ACP history popup. Notes Browse portal is a main-window AppView::NotesBrowseView { filter, selected_index } expanded two-pane portal, not the floating Notes editor. Persistent data/storage: SQLite notes table: id, title, Markdown content, timestamps, deleted state, pinned flag, sort order. SQLite note_cart_items: per-note staged context payloads with note-scoped deletion. FTS5 index/triggers for note search; fallback substring/LIKE behavior. Window bounds persisted under WindowRole::Notes, not main. Note content saves are debounced; dirty state/last-save confirmation drives footer feedback. Deleted notes are soft-deleted first; old Trash entries are pruned. Runtime caches/handles: Global Notes window handle/entity cache; cleared on close/drop. embedded_acp_chat: Option<Entity<AcpChatView>> on NotesApp, independent from ScriptListApp. notes_acp_generation: u64 rejects stale ACP actions after replacement. mention_portal_edit tracks local @note replacement sessions. history_back / history_forward, pending_focus_surface, action feedback timer, hover/focus flags, scroll handles. Notes Browse uses notes_browse_scroll_handle and reanchors selection from scroll state. Error/empty/loading states: First launch creates a welcome note if active and deleted notes are empty. Empty all-notes editor shows “No notes yet” plus create affordance. Empty Trash shows “Trash is empty” plus back-to-notes affordance. Notes Browse shows “No notes yet” for empty data and “No notes match your filter” for filtered no-match. Notes Browse search/storage errors log notes_browse_portal_load_failed and render empty rows rather than panic.
