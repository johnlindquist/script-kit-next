Non-negotiable invariants 3.1 Never log raw untrusted values Any log site interpolating stdin text, chat titles, dictation queries, triggerBuiltin names, ACP command displays, or similar untrusted values must use the shared safe-value helpers. The preview cap is bytes, not characters, defaults to 200 bytes, preserves UTF-8 boundaries, trims trailing whitespace, and appends … inside the byte budget. The rate limiter buckets by category, key length, and hash so raw untrusted strings are not retained; its default window is 30 seconds, stale buckets are garbage-collected after 120 seconds, and the map is capped at 2048 keys. logging-diagnostics-observabili… Every such log must include *_preview = %safe, *_bytes = safe.raw_bytes, *_safe_bytes = safe.safe_bytes, *_truncated = safe.truncated, and suppressed = rate.suppressed. That byte metadata is part of the contract, not decoration. logging-diagnostics-observabili… 3.2 Logs must be compact by default, verbose on demand Compact stderr format is SS.mmm|L|C|message, where L is severity and C is component category. Compact mode is enabled with SCRIPT_KIT_AI_LOG=1; full debug mode uses RUST_LOG=debug; module-specific debug can use a targeted RUST_LOG, such as RUST_LOG=script_kit::theme=debug. logging-diagnostics-observabili… dev.sh defaults SCRIPT_KIT_AI_LOG to 1, prints the chosen log mode on startup, and allows override with SCRIPT_KIT_AI_LOG=0 ./dev.sh or RUST_LOG=debug ./dev.sh. logging-diagnostics-observabili… logging-diagnostics-observabili… 3.3 Prefer machine-readable diagnostics over grep Protocol stats are exposed through kit://diagnostics/protocol-stats as a live JSON resource over Rust↔Bun boundary counters. The response includes snapshot, health, and thresholds; health flags are stable because the health function walks counters in declaration order. Zero-tolerance thresholds apply to stdin parse failures, too-large commands, and unsupported protocol versions, while expected traffic has higher thresholds: 10 unknown triggerBuiltin typos and 10,000 deprecated-name calls. logging-diagnostics-observabili… Transaction traces are also machine-readable: kit://transactions/latest and kit://transactions/schema expose the most recent transaction execution and its schema so agents do not need to tail logs manually. The latest reader can return a pretty-printed trace or an explicit empty payload with kind: "transactionTrace", status: "empty", and a “No transaction traces found” message. logging-diagnostics-observabili… logging-diagnostics-observabili… 3.4 Bounded persistence and corruption recovery are required Transaction trace persistence is bounded and tolerant of malformed JSONL. The trace layer appends schema-versioned traces, streams for the latest matching trace, and compacts by rewriting retained valid entries only. The implementation caps transaction traces at 10 MiB and keeps up to 2,000 compacted entries. logging-diagnostics-observabili… logging-diagnostics-observabili… AI preflight audits have the same operational shape: bounded JSONL, malformed-line tolerance, schema-drift tolerance, and correlation-level dedupe. The default path is ~/.scriptkit/logs/ai-preflight-audits.jsonl; schema version is 2; the max size is 5 MiB; and compaction keeps up to 2,000 records. logging-diagnostics-observabili… logging-diagnostics-observabili… 4. The two required controls are byte-capped previews and time-window rate limiting. Its public entry points are standalone waitFor and transactional batch; both carry request_id, success/failure, elapsed time, typed errors, and optional traces. logging-diagnostics-observabili… logging-diagnostics-observabili… Trace inclusion is controlled by TransactionTraceMode: On includes traces, OnFailure includes them only on failure, and Off omits them. The persistence helper calls append_transaction_trace only when the mode policy says to include the trace. logging-diagnostics-observabili… logging-diagnostics-observabili… Idempotency is enforced by request_id plus stable command fingerprint. Acceptance criteria for Feature 039 changes A change in this feature is acceptable only if it satisfies the relevant subset below: Privacy and budget: no raw untrusted value is logged; safe preview metadata is present; burst spam is rate-limited. Machine-readable: operator-critical diagnostics are exposed as JSON/MCP resources or structured records, not only prose log lines.
