Overlap cautions Keep these cautions explicit in machine metadata and boundary states: Main menu, launcher, footer, prompt, and popup claims are not Feature 039 claims. Feature 039 can show logs or transaction receipts around them, but the owning surface must still provide getState, getElements, surface contracts, action receipts, or runtime UI proof. ACP/AI context behavior is adjacent, not owned here. Feature 039 owns the preflight audit ledger and correlation proof; ACP/context features own whether the UI and context resolution behavior are correct. Protocol automation is adjacent. Feature 039 models waitFor/batch traces and replay identity, but protocol features own message semantics, targeting, and runtime command behavior. Screenshots are not the default proof. State-first receipts and MCP resources come first; screenshots are only appropriate when the failure is visual. Related features should remain cross-linked, not absorbed: Feature 004 for MCP/context/protocol automation, 030/031 for AI/chat APIs, 038 for agent skills/context catalog, and 040 for surface contracts. The Feature 039 chapter lists those as related rather than owned. feature-xstate-observability Bottom line: make the machine rich enough that the explorer can distinguish compact logs, safe logging, rate limiting, stable debug markers, protocol health, transaction execution, trace/resource/replay behavior, AI preflight audit persistence, and “observability only, not UI proof” boundaries. Do not make it a happy-path log viewer. The regression and gap states are part of the feature.
