Agent-first

Don't learn the settings. Ask.

Scripts, not extensions

Ten lines of TypeScript.
Or zero — the agent writes it.

A Script Kit script is one TypeScript file in ~/.scriptkit. await arg() gives you a full launcher UI for free, and the script is runnable the second you save — no build step, no store review.

And you don't have to write it yourself. Agents author scripts through the built-in MCP server (kit/scripts_create) the same way they edit your config: describe the chore, get a keystroke.

Deep links everywhere

Every corner of the app has a URL.

Every command, script, note, chat thread, and clipboard entry has an address. Notes and Day Pages embed live kit:// references — click one and the real thing reopens, with provenance, not a stale pasted copy.

And because commands map 1:1 to scriptkit:// deep links, you can bind any of them to a global shortcut in config.ts — any surface of the app, one custom keystroke away, from anywhere on your Mac.

Free · Open source · Native

Rust + GPUI. Plain files. No lock-in.

Script Kit is free and open source, written in Rust on GPUI — the GPU-accelerated UI framework behind Zed. No web view, no bundled browser: every list, every keystroke, every frame is native code, and it feels like it.

Everything it stores — scripts, notes, brain, themes, config — is a plain file in your home directory you can grep, sync, and version. The built-in MCP server exposes all of it to Claude, Codex, and any other agent. No account, no cloud lock-in.
